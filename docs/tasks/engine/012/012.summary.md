# Task 012 Summary: Split Asteroids Into Menu, Options, Highscores, and Game Scenes

## Goal

Split `playground-2d-asteroids` from one all-in-one scene into focused scenes:

- `main-menu`
- `options`
- `highscores`
- `game`

The gameplay scene should stop owning menu/options/highscores routing. This is a cleanup after Task 011, not a new Asteroids-specific engine layer.

## Main Work

- [ ] Add generic session-level state for values that survive scene transitions.
- [ ] Add Rhai `world.session` API.
- [ ] Move menu UI/script to `scenes/main-menu`.
- [ ] Move LOW mode to `scenes/options`.
- [ ] Move highscores UI/script to `scenes/highscores`.
- [ ] Move gameplay entities and gameplay script to `scenes/game`.
- [ ] Update scene routing with `world.scene.select(...)`.
- [ ] Update tests and startup config.

## Key Engine Requirement

`SceneStateService` is scene-local and should keep resetting on scene reloads. Add a separate generic `SessionStateService` for:

- `asteroids.low_mode`
- `asteroids.highscore.1`
- `asteroids.highscore.2`
- `asteroids.highscore.3`
- `asteroids.highscore.4`
- `asteroids.highscore.5`

Do not add `AsteroidsState`.

## UI Visibility Requirement

Do not rely only on entity lifecycle for UI documents.

Use root UI paths:

```rhai
world.ui.show("playground-2d-asteroids-main-menu.root");
world.ui.hide("playground-2d-asteroids-main-menu.root");
```

Tests must assert `UiStateService::is_visible("...root")`.

## Definition Of Done

- [ ] Game starts at `main-menu`.
- [ ] `START` routes to `game`.
- [ ] `OPTIONS` routes to `options`.
- [ ] `HIGHSCORES` routes to `highscores`.
- [ ] `Escape`/`BACK` returns to `main-menu`.
- [ ] LOW persists into gameplay.
- [ ] Highscores persist across scene transitions during the runtime session.
- [ ] `game/scene.rhai` has no menu/highscores/options branches.
- [ ] Menu overlay disappears when gameplay starts.
- [ ] No new Asteroids-specific engine systems.
