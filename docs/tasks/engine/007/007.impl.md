# TASK 007 - Procedural Vector Asteroids Foundations

## Status

Completed

## Goal

Build the first vector-first 2D playground for an Asteroids-style game without introducing a genre-specific engine feature.

The first slice is intentionally small:

1. new neutral `vector_2d` domain
2. scene-driven procedural vector shapes
3. WGPU stroke/fill rendering
4. new `playground-2d-asteroids` preview scene
5. no world wrap yet

## Current Collision Ownership

The current Asteroids preview now uses a hybrid ownership model.

That means:

1. overlap detection has moved into a reusable lightweight runtime collider seam
2. asteroid hit resolution and split outcomes still live in scene scripting
3. the current slice proves vector rendering, entity growth, audio hooks, and a first collider path before a fuller collision runtime

This is acceptable for the current milestone. Collision detection is now owned by reusable runtime queries, while hit/split outcomes intentionally stay scene-driven for this bounded MVP.
