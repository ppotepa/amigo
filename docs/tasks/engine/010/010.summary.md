## Intent

Przebudować `playground-2d-asteroids` tak, aby:

1. `scene.rhai` był cienką warstwą reguł gry,

## Agent 8 Integration Review - 2026-04-30

Current Asteroids migration status:

- [x] Runtime smoke coverage exists for `playground-2d-asteroids/vector-preview` bootstrap and interactive ship/fire/reload loop.
- [x] `scene.rhai` no longer uses `offscreen = 10000.0` as hidden state.
- [x] `scene.rhai` no longer calls manual `first_overlap_index(...)`; YAML collision event rules publish bullet/ship asteroid overlaps.
- [x] `scene.rhai` no longer defines local `generate_asteroid_points`; asteroid vectors are set through the generic vector API.
- [x] HUD updates now use state keys plus `world.ui.set_many(...)` instead of repeated per-field `set_text(...)`.
- [x] Bullets use generic entity pools/projectile emitters for acquire/fire/release.
- [ ] `scene.rhai` still owns asteroid name/alive/position/velocity/spin arrays and wave-specific activation.
- [ ] `scene.rhai` still owns ship kinematics: velocity, angle, angular velocity, acceleration, damping, speed clamp, and arena bounce.
- [ ] Bullet expiry still uses per-bullet script timers because declarative `Lifetime` starts ticking at scene load and is not reset on pool acquire.
- [ ] Generic `Velocity2d`, `Bounds2d`, and freeflight domain functions exist, but scene schema/runtime wiring is not complete enough for Asteroids to declare ship/asteroid movement and bounds behavior.
