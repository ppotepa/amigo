# TASK 013: Add Generic Value Curves and 2D Particle Emitters

## Objective

Introduce reusable nonlinear value mapping and generic 2D particle FX:

- `Curve1d` in `amigo_math`.
- Motion response curves for `FreeflightMotion2D`.
- `amigo_2d_particles` as a standalone 2D FX crate.
- Scene YAML, runtime, renderer, and Rhai integration.
- Asteroids main thruster implemented via `ParticleEmitter2D`.

No engine code may contain Asteroids-specific behavior.

## Phase A: Value Curves

Add to `crates/foundation/math`:

```rust
pub struct CurvePoint1d {
    pub t: f32,
    pub value: f32,
}

pub enum Curve1d {
    Constant(f32),
    Linear,
    EaseIn,
    EaseOut,
    EaseInOut,
    SmoothStep,
    Custom { points: Vec<CurvePoint1d> },
}
```

Rules:

- `sample()` clamps `t` to `0..1`.
- `Custom` sorts defensively during sampling.
- Empty custom curve returns `1.0`.
- One-point custom curve returns that value.
- Interpolation is linear.

Tests:

- linear, constant, ease-in, ease-out, smoothstep endpoints.
- custom interpolation.
- custom clamps before/after endpoints.
- unsorted custom points.
- empty custom points.

## Phase B: FreeflightMotion2D Response Curves

Extend `FreeflightMotionProfile2d`:

```rust
pub thrust_response_curve: Curve1d,
pub reverse_response_curve: Curve1d,
pub strafe_response_curve: Curve1d,
pub turn_response_curve: Curve1d,
```

Default all curves to `Curve1d::Linear`.

Modify `step_freeflight_motion_2d`:

- Clamp input to `-1..1`.
- Sample response curve using `abs(input)`.
- Reapply sign.
- Keep existing damping/max speed semantics unchanged.

Important: this affects analog magnitude response. It does not add temporal ramping for keyboard input.

## Phase C: Scene YAML Curves

Add scene document types:

```rust
#[serde(tag = "kind", rename_all = "snake_case")]
pub enum Curve1dSceneDocument {
    Constant { value: f32 },
    Linear,
    EaseIn,
    EaseOut,
    EaseInOut,
    SmoothStep,
    Custom { points: Vec<CurvePoint1dSceneDocument> },
}
```

Add optional fields to `FreeflightMotion2D`:

```yaml
thrust_response_curve:
  kind: ease_out
reverse_response_curve:
  kind: ease_in
strafe_response_curve:
  kind: linear
turn_response_curve:
  kind: smooth_step
```

Hydration defaults missing curves to linear.

Tests:

- old YAML without curves still hydrates.
- YAML with curves hydrates expected `Curve1d` values.

## Phase D: Particles Crate

Add workspace crate:

```text
crates/2d/particles
```

Library name:

```text
amigo_2d_particles
```

Core public types:

- `Particle2dPlugin`
- `Particle2dSceneService`
- `ParticleEmitter2d`
- `Particle2d`
- `Particle2dDrawCommand`
- `ParticleShape2d`
- `Particle2dEmitterRuntimeInput`

Initial renderer scope:

- Support `Circle { segments }`.
- Keep `Quad` and `Line` as model variants only if cheap, otherwise defer.

Runtime behavior:

- Inactive emitters do not spawn.
- Existing particles age and expire.
- Spawn rate is `spawn_rate * emission_rate_curve.sample(intensity)`.
- Size and alpha use age-normalized curves.
- Speed uses intensity curve plus jitter.
- `max_particles` caps live particles.
- Deterministic RNG seeded by emitter identity or explicit seed.

Tests:

- active emitter spawns particles.
- inactive emitter does not spawn.
- particles expire.
- max particle cap.
- size curve grows.
- alpha curve fades.
- parent velocity inheritance.

## Phase G: Rhai API

Add `world.particles`:


- `stop(entity)`

Defer `burst(entity, count)`.

## Phase H: Asteroids Integration

Replace static thrust triangle with:

```yaml
- type: ParticleEmitter2D
  attached_to: playground-2d-asteroids-ship
  local_offset: { x: -15.0, y: 0.0 }
  local_direction_degrees: 180.0
  active: false
  spawn_rate: 90.0
  max_particles: 96
  particle_lifetime: 0.34
  lifetime_jitter: 0.08
  initial_speed: 92.0
  speed_jitter: 36.0
  spread_degrees: 26.0
  inherit_parent_velocity: 0.20
  initial_size: 2.0
  final_size: 11.0
  color: "#F8F8F8DD"
  z_index: 38.0
  shape:
    kind: circle
    segments: 8
  emission_rate_curve:
    kind: ease_out
  size_curve:
    kind: ease_out
  alpha_curve:
    kind: custom
    points:
      - { t: 0.0, value: 0.90 }
      - { t: 0.60, value: 0.42 }
      - { t: 1.0, value: 0.00 }
  speed_curve:
    kind: ease_out
```

Rhai controls only emitter active/intensity. It must not manually sync emitter transforms.

## Non-Goals

- No Asteroids-specific engine systems.
- No per-particle `SceneEntity`.
- No particle collision.
- No particle burst/explosion requirement in first implementation.
- No input temporal ramping in this task.
