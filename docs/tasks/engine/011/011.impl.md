# TASK 011: Redesign `playground-2d-asteroids` as a Retro Vector Arcade Mini-Game

## Goal

Transform `playground-2d-asteroids` from a technical playground into a complete small arcade game inspired by classic Asteroids.

The current mod already has useful generic foundations:

- `collision_events` for bullet-asteroid and ship-asteroid events
- `EntityPool` for bullets
- `Velocity2D`
- `Bounds2D`
- `Lifetime`
- `FreeflightMotion2D`
- `UiDocument`
- `text_bind`

The remaining problem is design and orchestration. The scene still feels preset/test-driven: hardcoded asteroid names, `asteroid_alive`, manual wave 2, fixed `ship_start_x = -540.0`, short shield, debug HUD, and grid-heavy background.

This task should make the mod a real retro vector arcade mini-game while keeping the engine work agnostic. Do not add an `AsteroidsEngine`, `AsteroidsSystem`, or game-specific runtime feature.

## Chapter 1: Vision

The game should look and feel like retro vector arcade, not a debug showcase.

Target style:

- black background
- white/light-gray vector lines
- minimal color
- outlined ship
- irregular procedural asteroid polygons
- bullets as small dots/short lines
- shield as a thin circle
- small HUD, no large panel
- no debug grid in normal mode
- thrust visual/audio can be disabled for the first redesign pass

Design target:

```text
ASTEROIDS
> START <
  HIGHSCORES
  LOW: OFF
  QUIT

black background
ship starts at center
3s shield after spawn/respawn
procedural waves
vector asteroids
minimal HUD
```

## Chapter 3: Game Flow

Introduce explicit game modes:

```rhai
let game_mode = "menu";
```

Allowed values:

```text
menu
highscores
playing
wave_clear
respawn
game_over
quit_pending
```

Transitions:

```text
BOOT
  -> menu

menu
  START       -> start_new_game()
  HIGHSCORES  -> highscores
  LOW         -> toggle_low_mode()
  QUIT        -> quit_pending or app.quit()

playing
  all asteroids destroyed -> wave_clear
  ship hit without shield -> respawn or game_over

wave_clear
  after delay -> next wave

respawn
  player placed at center
  shield 3s
  -> playing

game_over
  Enter/Space -> menu or start_new_game()
```

Skeleton:

```rhai
let game_mode = "menu";
let menu_index = 0;
let menu_items = ["START", "HIGHSCORES", "LOW", "QUIT"];

let score = 0;
let lives = 3;
let wave_index = 1;
let low_mode = false;
let run_seed = 1337;

fn on_enter() {
    enter_menu(world);
}

fn update(dt) {
    if game_mode == "menu" {
        update_menu(world);
        return;
    }

    if game_mode == "highscores" {
        update_highscores(world);
        return;
    }

    if game_mode == "playing" {
        update_playing(world, dt);
        return;
    }

    if game_mode == "wave_clear" {
        update_wave_clear(world);
        return;
    }

    if game_mode == "respawn" {
        update_respawn(world);
        return;
    }

    if game_mode == "game_over" {
        update_game_over(world);
        return;
    }
}

fn on_event(topic, payload) {
    if game_mode != "playing" && game_mode != "respawn" {
        return;
    }

    if topic == "asteroids.bullet_hit_asteroid" {
        handle_bullet_hit(world, payload[0], payload[1]);
        return;
    }

    if topic == "asteroids.ship_hit_asteroid" {
        handle_ship_hit(world);
        return;
    }
}
```

```rhai
        shield.hide();

```

Ship hit:

```rhai
fn handle_ship_hit(world_api) {
    if game_mode != "playing" {
        return;
    }

    if world_api.timers.active("ship-shield") {
        return;
    }

    lives -= 1;
    sync_hud(world_api);

    if lives <= 0 {
        enter_game_over(world_api);
        return;
    }

    game_mode = "respawn";
    set_status(world_api, "READY");
    reset_player(world_api);
}
```

- shield always lasts `3.0s`
- shield protects gameplay, not only visuals
- respawn gives shield
- no ship-hit handling in menu/highscores/game-over

## Chapter 11: Procedural Waves

No separate maps. A level is:

```text
run + wave_index + difficulty parameters
```

Wave parameters:

```rhai
fn wave_asteroid_count(wave) {
    let base = 3 + wave;
    if low_mode {
        base -= 1;
    }
    return clamp_int(base, 3, 16);
}

fn wave_speed_min(wave) {
    let value = 55.0 + wave * 8.0;
    if low_mode {
        return value * 0.72;
    }
    return value;
}

fn wave_speed_max(wave) {
    let value = 110.0 + wave * 12.0;
    if low_mode {
        return value * 0.72;
    }
    return value;
}

fn wave_radius_min(wave) {
    return max_float(14.0, 34.0 - wave * 1.2);
}

fn wave_radius_max(wave) {
    return 44.0;
}
```

Rhai seeded RNG is acceptable for this task:

```rhai
let rng_seed = 1337;

fn rand01() {
    rng_seed = (rng_seed * 1664525 + 1013904223) % 4294967296;
    return rng_seed / 4294967296.0;
}

fn rand_range(min, max) {
    return min + (max - min) * rand01();
}

fn rand_int(min, max) {
    return floor(rand_range(min, max + 1.0));
}
```

Future engine feature `world.random` is optional and should be a separate generic improvement.

Spawn away from center:

```rhai
fn random_spawn_position_away_from_center() {
    let x = rand_range(-860.0, 860.0);
    let y = rand_range(-460.0, 460.0);

    if abs(x) < 220.0 && abs(y) < 180.0 {
        if abs(x) < abs(y) {
            if y >= 0.0 {
                y = 260.0;
            } else {
                y = -260.0;
            }
        } else {
            if x >= 0.0 {
                x = 300.0;
            } else {
                x = -300.0;
            }
        }
    }

    return #{ x: x, y: y };
}
```

Start wave:

```rhai
fn start_wave(world_api, wave) {
    world_api.pools.release_all("asteroids");

    let count = wave_asteroid_count(wave);
    let i = 0;

    while i < count {
        let asteroid = world_api.pools.acquire("asteroids");
        if asteroid == "" {
            return;
        }

        let radius = rand_range(wave_radius_min(wave), wave_radius_max(wave));
        let vertices = rand_int(8, 14);
        let speed = rand_range(wave_speed_min(wave), wave_speed_max(wave));
        let angle = rand_range(0.0, 6.2831853);
        let pos = random_spawn_position_away_from_center();

        world_api.entities.set_position_2d(asteroid, pos.x, pos.y);
        world_api.motion.set_velocity(asteroid, cos(angle) * speed, sin(angle) * speed);

        world_api.vector.set_radial_jitter_polygon(
            asteroid,
            vertices,
            radius,
            0.24,
            rng_seed + i
        );

        world_api.physics.set_circle_radius(asteroid, radius);
        world_api.entities.set_property_string(asteroid, "asteroid_size", asteroid_size_for_radius(radius));
        world_api.entities.set_property_int(asteroid, "score_value", score_for_radius(radius));

        world_api.entities.show(asteroid);
        world_api.entities.enable_simulation(asteroid);
        world_api.entities.enable_collision(asteroid);

        i += 1;
    }

    wave_index = wave;
    sync_hud(world_api);
    set_status(world_api, "WAVE " + pad_two(wave));
    world_api.audio.play("wave-start");
}
```

Required API if missing:

```text

world.pools.release("asteroids", entity)
world.pools.release_all("asteroids")
world.pools.active_count("asteroids")

world.entities.set_property_string(entity, key, value)
world.entities.set_property_int(entity, key, value)
```

If missing, add generic APIs. Do not return to hardcoded arrays.

## Chapter 12: Collision Events And Scoring

```rhai
fn handle_bullet_hit(world_api, bullet, asteroid) {
    world_api.pools.release("player-bullets", bullet);
```
