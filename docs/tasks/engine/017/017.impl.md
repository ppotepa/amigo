# TASK 017 Implementation Plan: INK WARS

## 0. Kontekst

`INK WARS` to kompletny testowy mod dla Amigo Engine: lokalny split-screen 2D duel z niszczalnym terenem, projectile-heavy broniami, pickupami i UI per player.

Najważniejsza zmiana względem wcześniejszego planu: primary visual target to szkolny zeszyt i niebieski długopis. Terrain, postacie, HUD, particles i tła mają wyglądać jak proceduralnie rysowany szkic. To wymusza nowy renderer `ink_render_2d`, a nie tylko zmianę palety.

Punkt wyjścia engine:

- istnieją `motion_2d`, `physics_2d`, `particles_2d`, `vector_2d`, UI i service-based runtime;
- istnieje prosty projectile launcher w `motion_2d`;
- istnieją Rhai packages i cienkie wrappery gameplayowe;
- obecny prototyp moda może zostać przemianowany/migrowany z `ink-wars`.

## 4. Architecture Rule

Gameplay i render muszą być rozdzielone.

Gameplay terrain:

```text
solid mask
damage brush
raycast
circle cast
collision sampling
dirty region
dirty chunks
```

Visual terrain:

```text
marching squares
contour loops
surface classification
hatch regions
deterministic details
ink commands
```

Nie wolno:

- renderować stylu jako jednego static background PNG;
- losować jittera co frame;
- mieszać ink renderer z physics;
- robić particles jako głównego źródła gameplay collision;
- dodawać Worms/Liero-specific typów do engine.


Odpowiedzialność:

- notebook lines;
- margin/spiral holes;

- hatch/crosshatch;
- doodle primitives;
- ink particles bridge;
- ink UI skin;

Model:

```rust
pub struct InkRenderCommand2d {

```

Wymaganie krytyczne:



## 8. Terrain Ink Material

Terrain nie używa klasycznego `surface_texture/interior_texture` jako primary renderer.

Używa:

- outline;
- crater outline;
- top surface style;
- wall/cave style;
- hatch/crosshatch style;
- grass/pebble/crack detail rules.

Przykładowy model:

```rust
pub struct TerrainInkMaterial2d {
    pub id: String,
    pub outline: InkStrokeStyle2d,
    pub crater_outline: InkStrokeStyle2d,
    pub top_surface: TerrainInkSurfaceStyle2d,
    pub wall_surface: TerrainInkSurfaceStyle2d,
    pub cave_surface: TerrainInkSurfaceStyle2d,
    pub hatch: TerrainHatchStyle2d,
    pub cross_hatch: Option<TerrainHatchStyle2d>,
    pub detail_rules: Vec<TerrainDetailRule2d>,
}
```

## 9. Terrain Visual Extraction

`destructible-terrain` generuje:

```rust
pub struct TerrainVisualData2d {
    pub terrain_entity: String,
    pub contours: Vec<TerrainContourLoop2d>,
    pub surface_segments: Vec<TerrainSurfaceSegment2d>,
    pub hatch_regions: Vec<TerrainHatchRegion2d>,
    pub details: Vec<TerrainDetailInstance2d>,
    pub dirty_regions: Vec<TerrainDirtyRegion2d>,
}
```

`ink-render` konwertuje:

```text
TerrainVisualData2d + TerrainInkMaterial2d + NotebookInkStyle2d
  -> Vec<InkRenderCommand2d>
```

Wygenerowane commands:

- outer contour strokes;
- cave contour strokes;
- crater contour strokes;
- hatch fills;
- crosshatch fills;
- grass doodles;
- pebble doodles;
- cracks;
- debris near dirty regions;
- smoke/impact ink details.
