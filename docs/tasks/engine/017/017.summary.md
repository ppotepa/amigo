# TASK 017: INK WARS / ink-wars

## Cel

Zbudować kompletny mod `INK WARS`: lokalny split-screen 2D terrain duel z niszczalnym terenem, broniami projectile/particle, pickupami, HUD per player i pełnym flow `menu -> match -> result`.

Najważniejszy kierunek artystyczno-techniczny: gra ma wyglądać jak narysowana niebieskim długopisem w szkolnym zeszycie. Notebook/ink rendering nie jest dodatkiem. To primary renderer i główna motywacja techniczna taska.

## Nazwy

- tytuł gry: `INK WARS`
- docelowy mod id: `ink-wars`
- obecny prototyp może być migrowany z `ink-wars`
- launcher label: `INK WARS`

## Główne fundamenty engine

Wymagane domeny:

- `destructible_terrain_2d`: gameplayowa maska terenu, damage, dirty chunks, collision/query, contour extraction i visual data.
- `ink_render_2d`: papier, linie zeszytu, margines, spiral, jittered strokes, hatching, doodles, ink UI skin i ink particle bridge.
- `viewport_2d`: split screen, wiele kamer, per-player HUD.

Bardzo rekomendowane domeny:

- `projectile_2d`: runtime pocisków, lifetime, bounce, impact, explosion.
- `pickup_2d`: spawn/respawn/effect lifecycle dla znajdziek, jeśli pickup flow zacznie wychodzić poza prosty script-only model.

## Najważniejsza decyzja

Gameplay i render są rozdzielone.

Terrain gameplay:

- solid mask;
- raycast/circle cast;
- damage brushes;
- collision sampling;
- dirty regions/chunks.

Terrain visual:

- marching squares;
- contour loops;
- surface classification;
- hatch/crosshatch regions;
- deterministic details;
- conversion to ink render commands.

Nie renderować notebook stylu jako jednego statycznego PNG. Terrain jest niszczalny, więc kontury, kreski i detale muszą wynikać z aktualnej maski.
