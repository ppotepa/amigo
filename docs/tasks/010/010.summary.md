## Intent

Przebudować `playground-2d-asteroids` tak, aby:

1. `scene.rhai` był cienką warstwą reguł gry,
2. powtarzalna mechanika (motion, bounds, pooling, collisions, timers, state, UI binding, procedural vector) była reusable API engine,
3. nie powstał żaden Asteroids-specific subsystem w core,
4. nowe API było bardziej elastyczne także dla przyszłych modów, a nie tylko dla Asteroids.

## Architecture Guardrails

- [ ] Brak typów/nazw `Asteroids*` w engine
- [ ] Feature'y są plugin-first i SRP
- [ ] Rhai bindings są modułowe (bez monolitu)
- [ ] YAML-first, Rhai-thin
- [ ] Czysta logika domenowa testowalna bez runtime, gdzie możliwe
- [ ] API wspiera zarówno ścieżkę deklaratywną, jak i imperatywną tam, gdzie to ma sens
- [ ] Nie mnożyć crate'ów bez wyraźnej granicy capability
- [ ] Nie wprowadzać zbyt wąskich modeli, które zaraz wymuszą kolejny breaking cleanup
