# TASK 014: HUD/UI Showcase + Runtime Themes + First Controls

## Cel

Dodać generyczne fundamenty UI potrzebne do przyszłych narzędzi developerskich i particle editora:

- runtime themes;
- style classes dla node'ów UI;
- przełączanie theme bez reloadu sceny;
- pierwsze lepsze kontrolki: slider i toggle;
- showcase mod `playground-hud-ui`, który udowadnia HUD, menu, panele, controls i theming.

Ten task jest dependency dla `015 playground-2d-particles`, bo particle editor potrzebuje realnych kontrolek UI zamiast protez na przyciskach.

## Zakres

### Engine/UI

- Dodać `UiTheme`, `UiThemePalette`, `UiThemeService`.
- Dodać `style_class: Option<String>` do `UiNode`.
- Dodać `UiThemeSet` jako komponent sceny YAML.
- Dodać `world.ui.set_theme(theme_id)` i opcjonalnie `world.ui.theme()`.
- Dodać resolving stylu w kolejności:
  1. `UiStyle::default()`
  2. theme class style
  3. node inline style
  4. runtime overrides z `UiStateService`
- Dodać `UiNodeKind::Slider`.
- Dodać `UiNodeKind::Toggle`.
- Dodać event `on_change` dla slider/toggle.
- Rozszerzyć UI input o drag slidera i toggle click.

### Mod

Dodać:

```text
mods/playground-hud-ui/
  mod.yml
  scenes/showcase/scene.yml
  scenes/showcase/scene.rhai
```

Showcase ma pokazywać:

- top bar z theme switcherem;
- left controls panel;
- HUD cards;
- editor panel preview;
- notification/toast;
- bottom debug bar.

Theme:

- `space_dark`
- `clean_dev`

Sterowanie:

- `F1` -> `space_dark`
- `F2` -> `clean_dev`
- `T` -> cycle theme

## Definition of Done

- `world.ui.set_theme("space_dark")` działa ze skryptu.
- Theme zmienia kolory UI bez reloadu sceny.
- `style_class` działa w YAML node'ach.
- Runtime overrides nadal mają priorytet nad theme.
- Slider działa myszką przez drag i publikuje `on_change`.
- Toggle działa kliknięciem i publikuje `on_change`.
- `playground-hud-ui/showcase` bootstrappuje się i pokazuje oba theme.
- Testy pokrywają theme switching, style resolving, slider input, toggle input i Rhai API.
