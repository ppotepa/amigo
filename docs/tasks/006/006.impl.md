# TASK 006 - Platformer to Motion Compatibility Migration

## Status

In progress

## Type

Architecture / Naming / Compatibility Migration

## Priority

High

## Goal

Replace `platformer` as a first-class engine concept with a neutral `motion` layer, without breaking the existing runtime, Rhai API, scene hydration, diagnostics, or `playground-sidescroller`.

This task is explicitly **compatibility-first**:

1. do not start with renaming the crate folder
2. do not start by deleting `world.platformer`
3. do not start by changing YAML schema first

The migration should introduce neutral naming first, keep aliases working, and only perform hard cleanup at the end.

## Confirmed Dependency Map

The current `platformer` surface still appears in:

1. `crates/2d/platformer`
2. `bootstrap` plugin registration
3. diagnostics / summary
4. `systems/motion_2d.rs`
5. `scene_runtime` context + handler
6. `engine/scene` YAML schema + hydration + scene commands + scene events
7. `crates/scripting/rhai` runtime constructor and bindings
8. `mods/playground-sidescroller/mod.toml`
9. `mods/playground-sidescroller/scenes/vertical-slice/scene.rhai`
10. app and Rhai tests

## Core Rule

Prefer this sequence:

1. add neutral aliases
2. add neutral capability
3. add neutral Rhai API
4. switch internal users
5. switch playground content
6. only then remove or rename old terms

## Planned Delivery

### Phase 1 - Neutral Aliases in Existing Crate

Add aliases inside `crates/2d/platformer`:

1. `Motion2dPlugin`
2. `Motion2dSceneService`
3. `MotionProfile2d`
4. `MotionState2d`
5. `Facing2d`
6. `MotionAnimationState`

Keep old `Platformer*` names working.

Implemented:

1. `Motion2dSceneService`
2. `Motion2dPlugin`
3. `MotionProfile2d`
4. `MotionState2d`
5. `MotionIntent2d`
6. `MotionController2d`
7. `MotionController2dCommand`
8. `MotionDriveResult`
9. `Facing2d`
10. `MotionAnimationState`
11. `drive_motion_2d(...)`
12. `motion_animation_state_for(...)`

### Phase 2 - Capability Compatibility

Introduce `motion_2d` while preserving compatibility for `platformer_2d`.

Goal:

1. diagnostics can report neutral capability
2. existing tests and mod capability requirements can keep passing during migration

Implemented:

1. `Motion2dDomainInfo`
2. registration of `motion_2d` alongside `platformer_2d`
3. summary/diagnostics now include the neutral capability without removing the old one

### Phase 3 - Rhai Compatibility

Add:

1. `world.motion`

Keep:

1. `world.platformer`

Both should delegate to the same underlying service during migration.

Implemented:

1. `world.motion`
2. `WorldMotion` / `MotionState` Rhai type names
3. compatibility aliasing through the same underlying backend as `world.platformer`
4. direct Rhai test coverage proving `world.motion.state(...)` and `world.motion.drive(...)` work

### Phase 4 - Internal Runtime Naming

Switch internals toward neutral naming:

1. systems
2. services
3. handler naming
4. summary/diagnostics wording where safe

Do not break scene hydration or mod content yet.

Implemented:

1. runtime tick entrypoint now uses `tick_motion_2d_world(...)`
2. compatibility wrapper `tick_platformer_world(...)` remains available for now

### Phase 5 - Playground Migration

Migrate `playground-sidescroller`:

1. script from `world.platformer.*` to `world.motion.*`
2. mod capability requirement from `platformer_2d` to `motion_2d`

Keep backward compatibility in code until tests are stable.

Implemented:

1. `mods/playground-sidescroller/mod.toml` now requires `motion_2d`
2. `mods/playground-sidescroller/scenes/vertical-slice/scene.rhai` now uses `world.motion.*`
3. compatibility alias `world.platformer` remains in engine code
4. app/runtime/bootstrap and Rhai test suites stay green after the migration checkpoint

### Phase 6 - Schema and Command Cleanup

Only after stability:

1. `PlatformerController2D` -> neutral scene component name
2. `QueuePlatformerController2d` -> neutral scene command name
3. `SceneEvent::PlatformerControllerQueued` -> neutral event

Implemented:

1. `MotionController2D` is now the primary YAML component name
2. `PlatformerController2D` still parses as a compatibility alias
3. hydration now emits `QueueMotionController2d`
4. runtime still accepts legacy `QueuePlatformerController2d`
5. scene runtime now publishes `SceneEvent::MotionControllerQueued`

### Phase 7 - Hard Rename

Last step only:

1. `crates/2d/platformer` -> `crates/2d/motion`
2. `amigo-2d-platformer` -> `amigo-2d-motion`
3. update workspace dependencies/import paths

## Non-Goals

This task does not yet include:

1. asset processor registry
2. plugin bundles
3. scene hydrator registry
4. capability registry
5. full removal of compatibility aliases in the same pass

## Acceptance

This task is done when:

1. `world.motion` exists and works
2. `playground-sidescroller` uses neutral motion API
3. neutral capability `motion_2d` is present
4. old `platformer` compatibility path can be removed or clearly deprecated
5. tests remain green throughout the staged migration
