## Current Progress

- `script_runtime` zostało rozbite na moduły handlerów per namespace.
- `process_ui_input` zostało wyciągnięte z `InteractiveRuntimeHostHandler` do `systems/ui_input.rs`.
- `host_runtime` używa bezpośrednio runtime `SystemRegistry` i współdzielonego `SystemPhase`; app-local `PhaseRunner` został usunięty.
- `amigo-runtime` ma już pierwszy generic scheduler registry (`SystemRegistry`) z testami.
- `amigo-app` rejestruje `ui_input` jako normalny runtime system w fazie `PreUpdate`; viewport dla hit-testu jest trzymany w runtime service `UiInputViewportState` zamiast lokalnego host state.
- `amigo-app` nie ma już centralnego `build_runtime_system_registry()`; rejestracja runtime systems została rozbita na mniejsze plugins o węższym zakresie.
- `amigo-app` w aktywnej ścieżce bootstrap/scene runtime/system przeszedł z `Platformer*` na `Motion*` surface tam, gdzie aliasy już istniały.
- `amigo-2d-platformer` ma już canonical `Motion*` source definitions, a `Platformer*` jest warstwą compatibility aliasów.
- runtime po `MOTION_2D_PLUGIN` publikuje już tylko canonical `Motion2dDomainInfo`; legacy `PlatformerDomainInfo` nie jest już rejestrowane jako aktywna capability.
- `amigo-scene` ma canonical motion-first helpers dla commands/events oraz legacy platformer jako compatibility path.
- `amigo-scripting-rhai` ma `motion` jako jedyną aktywną publiczną ścieżkę runtime/API dla tej domeny; `world.platformer` zostało usunięte z aktywnego API.
- diagnostics i summary biorą motion-first runtime report label z domeny motion zamiast ręcznego mapowania stringów w `app`.
- `amigo-app` nie trzyma już ręcznie literalnego legacy motion label w diagnostics/test expectations; używa canonical constant z domeny motion.
- `render-api` ma pierwszy `RenderFramePacket<TOverlay>`, a `ui_runtime` buduje UI overlay packet zamiast inline list w host render path.
- `render-api` ma już wspolny `RenderExtractorRegistry<TContext, TOverlay>`, a `ui_runtime` korzysta z niego jako pierwszego app-side extractor slice dla UI overlay.
- `render_runtime` w `amigo-app` zbiera już wspólny packet render data dla `TileMap2D`, `Sprite2D`, `Text2D`, `Vector2D`, `Mesh3D`, `Material3D`, `Text3D` i UI overlay, a host redraw nie przekazuje już runtime scene services tych domen bezpośrednio do renderera.
- `engine/scene` aktywnie wystawia już tylko canonical `QueueMotionController2d` i `MotionControllerQueued`; legacy platformer variants zostały usunięte z aktywnego enum surface.
- `cargo test -p amigo-app` przechodzi po tych zmianach.
