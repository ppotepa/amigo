## 2. Decyzja UI: TabView vs GroupBox

### Nazewnictwo

Nie nazywac zakladek `GroupBox`.

W typowej terminologii UI:

- `GroupBox` to labelled container: ramka + tytul + zawartosc.
- `TabView`, `TabbedPanel` albo `TabControl` to tab bar + aktywny panel.

Rekomendacja dla repo:

```text
UiNodeKind::TabView
UiNodeKind::GroupBox
```

Ale wdrazac etapami.

### Etap 1: kompozycja bez nowej kontrolki

Zakladki stosowac w scenie editor, nie w showcase. Showcase ma byc ekranem demonstracyjnym z dropdownem presetow i czytelnym preview. W pierwszym kroku editor moze uzyc istniejacych kontrolek:

```yaml
- type: column
  id: editor
  children:
    - type: option-set
      id: tabs
      selected: "Color"
      options: ["Presets", "Emission", "Color", "Size", "Motion", "Forces", "Spawn", "Burst", "Export"]
      on_change:
        event: playground-2d-particles.tabs.changed

    - type: stack
      id: tab-panels
      children:
        - type: column
          id: gallery-panel
        - type: column
          id: emission-panel
        - type: column
          id: color-panel
```

Rhai steruje widocznoscia paneli:

```rhai
fn tab_panel_path(name) {
    return "playground-2d-particles-editor-ui.root.body.main.tab-panels." + name + "-panel";
}

fn set_tab(world_api, tab) {
    world_api.state.set_string("selected_tab", tab);

    let tabs = ["presets", "emission", "color", "size", "motion", "forces", "spawn", "burst", "export"];
    let i = 0;
    while i < tabs.len() {
        if tabs[i] == tab.to_lower() {
            world_api.ui.show(tab_panel_path(tabs[i]));
        } else {
            world_api.ui.hide(tab_panel_path(tabs[i]));
        }
        i += 1;
    }
}
```

To da dzialajace zakladki bez zmian w engine.

### Dropdown popup layer

Dropdown po rozwinieciu ma dzialac jak popup z wyzszym priorytetem render/hit-test, a nie jak kontrolka o wiekszej wysokosci.

Wymagania:

```text
- expanded dropdown nie zmienia `measure_node`;
- siblings w Column/Row nie przesuwaja sie po rozwinieciu;
- lista opcji renderuje sie po normalnych primitive UI jako popup layer;
- hit-test dropdownu nadal obejmuje obszar listy poza bazowym rectem;
- klik w dalsza opcje nie moze byc zasloniety przez kontrolki pod spodem.
```

### Etap 2: natywne `TabView`

Dodac dopiero gdy kompozycja zacznie byc powtarzana w kilku modach.

Model runtime:

```rust
pub enum UiNodeKind {
    // existing...
    TabView {
        selected: String,
        tabs: Vec<UiTabItem>,
    },
    GroupBox {
        title: String,
    },
}

pub struct UiTabItem {
    pub id: String,
    pub label: String,
    pub disabled: bool,
}
```

Scene document:

```rust
pub enum SceneUiNodeKind {
    // existing...
    TabView {
        selected: String,
        tabs: Vec<SceneUiTabItemDocument>,
    },
    GroupBox {
        title: String,
    },
}
```

YAML:

```yaml
- type: tab-view
  id: particle-tabs
  selected: color
  style_class: tab_view
  on_change:
    event: playground-2d-particles.tabs.changed
  tabs:
    - { id: gallery, label: Gallery }
    - { id: emission, label: Emission }
    - { id: color, label: Color }
    - { id: forces, label: Forces }
  children:
    - type: column
      id: gallery
    - type: column
      id: emission
    - type: column
      id: color
    - type: column
      id: forces
```

Renderer/layout:

- tab bar height: 38-44 px;
- active tab gets `tab_active`;
- inactive tab gets `tab`;
- disabled tab gets `tab_disabled`;
- only active child panel participates in layout/hit-testing.

Acceptance:

```text
TabView publikuje on_change z selected tab id.
TabView zapisuje selected do UiStateService.
Nieaktywne tab panels nie lapia klikniec.
TabView mozna stylowac przez theme classes.
```

## 3. Docelowy layout `playground-2d-particles`

### 3.1. Showcase scene

Showcase ma jeden preset naraz w preview. Nie powinien miec prawego tabbed inspectora; szczegoly edycji naleza do sceny editor. World preview nadal ma byc jeden 400x400 grid z aktywnym emiterem na srodku.

Makieta:

```text
+--------------------------------------------------------------------------------+
| TOP BAR: PARTICLE SHOWCASE | selected=preset | [Menu]                           |
+--------------------------------------------------------------------------------+
| CENTER 400x400 PREVIEW                         | RIGHT CONTROL PANEL       |
|                                                | Preset: [Dropdown v]      |
|                   emitter center               | Intensity slider          |
|                    particles                   | Start / Stop / Reset      |
|                                                | Burst hint/status         |
+--------------------------------------------------------------------------------+
| FOOTER: 1-0 select | Space toggle | B burst | R reset | Esc menu                 |
+--------------------------------------------------------------------------------+
```

Najwazniejsza zasada layoutu: preview jest stabilne i zawsze 400x400. Zmienianie okna nie ma zmieniac logiki rozmieszczenia emitera.

Preset picker ma byc `Dropdown` / select input, nie `OptionSet`. Przy 10 presetach `OptionSet` jest jeszcze akceptowalny, ale przy docelowych 30+ zaczyna zajmowac zbyt duzo miejsca i utrudnia layout. `OptionSet` zostaje dla malych zamknietych wyborow, np. `spawn kind` albo `shape kind`.

Obecne referencje:

- preview world grid: `mods/playground-2d-particles/scenes/showcase/scene.yml`
- selected preset logic: `mods/playground-2d-particles/scenes/showcase/scene.rhai`
- UI viewport scaling: uzywac `viewport.scaling: fit` jak w obecnych scenach.

## 7. Brakujace API do dopisania

Status tej sekcji: API 7.1-7.5 zostalo zaimplementowane. Zostaje jako referencja implementacyjna i test-plan historyczny.

### 7.1. `set_max_particles`

Particles service:

```rust
pub fn set_max_particles(&self, entity_name: &str, max_particles: usize) -> bool {
    self.update_emitter(entity_name, |emitter| {
        emitter.max_particles = max_particles;
    })
}
```

Rhai:

```rust
pub fn set_max_particles(&mut self, entity_name: &str, value: rhai::INT) -> bool {
    if value <= 0 {
        return false;
    }
    self.particles
        .as_ref()
        .map(|particles| particles.set_max_particles(entity_name, value as usize))
        .unwrap_or(false)
}
```
