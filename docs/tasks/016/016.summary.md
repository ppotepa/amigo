# TASK 016: Advanced Particle Showcase Tabs + Authoring UX

## Cel

Przeksztalcic `playground-2d-particles` z prostego showcase/editor v0 w czytelny, zakladkowy particle authoring playground.

Ten task nie powtarza fundamentow juz dodanych w `014/015`: `amigo_fx::ColorRamp`, `ParticleEmitter2D.color_ramp`, `forces`, `spawn_area`, `burst`, podstawowy editor i UI controls juz istnieja.

## Najwazniejsza decyzja

Dla UI uzyc dwoch poziomow:

- krotkoterminowo: zakladki jako kompozycja `OptionSet` + panele `Stack/Column` sterowane przez Rhai;
- docelowo: dodac generyczna kontrolke `TabView` / `TabbedPanel`, a `GroupBox` traktowac jako osobny labelled container.

Nie nazywac tego `GroupBox with tabs`, bo to sa dwa rozne wzorce:

- `GroupBox`: ramka z tytulem dla sekcji formularza;
- `TabView` / `TabbedPanel`: kontener z tab bar i aktywnym panelem.

## Zakres

- Dodac zakladkowy layout do `mods/playground-2d-particles/scenes/showcase`.
- Wybieranie presetow zrobic jako `Dropdown` / select input, nie jako `OptionSet`, bo lista presetow ma rosnac do 30+.
- Rozbudowac editor o taby: `Gallery`, `Presets`, `Emission`, `Color`, `Size`, `Motion`, `Forces`, `Spawn`, `Burst`, `Export`.
- Dodac brakujace presety: `plasma`, `portal`, `explosion`, opcjonalnie `rain`.
- Przygotowac katalog presetow `mods/playground-2d-particles/presets/` jako przyszly source of truth.
- Dodac brakujace API tam, gdzie blokuje UX: `set_max_particles`, `set_color_ramp`, `set_shape`, export helper.
- Dopiero pozniej dodac natywne UI `TabView` i `GroupBox`.

## Definition of Done

- Showcase ma zakladki i nie jest jedna dluga kolumna kontrolek.
- Editor moze zmieniac emission/color/motion/forces/spawn/burst w odseparowanych panelach.
- Presety `fire`, `smoke`, `sparks`, `magic`, `snow`, `dust`, `thruster`, `plasma`, `portal`, `explosion` sa widoczne z UI.
- `Export` wypisuje YAML uwzgledniajacy `color_ramp`, `forces`, `spawn_area` i burst config.
- Dokumentacja wskazuje, ktore czesci sa kompozycja na obecnym UI, a ktore wymagaja natywnego `TabView`.
