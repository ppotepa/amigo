# TASK 016: Advanced Particle Showcase Select + Authoring UX

## Cel

Przeksztalcic `playground-2d-particles` z prostego showcase/editor v0 w czytelny particle authoring playground.

Ten task nie powtarza fundamentow juz dodanych w `014/015`: `amigo_fx::ColorRamp`, `ParticleEmitter2D.color_ramp`, `forces`, `spawn_area`, `burst`, podstawowy editor i UI controls juz istnieja.

## Najwazniejsza decyzja

Dla UI uzyc dwoch poziomow:

- showcase: prosty ekran demonstracyjny z `Dropdown`/select dla presetow i bez prawego tabbed inspectora;
- editor: zakladki jako kompozycja `OptionSet` + panele `Stack/Column` sterowane przez Rhai;
- docelowo: dodac generyczna kontrolke `TabView` / `TabbedPanel`, a `GroupBox` traktowac jako osobny labelled container.

Nie nazywac tego `GroupBox with tabs`, bo to sa dwa rozne wzorce:

- `GroupBox`: ramka z tytulem dla sekcji formularza;
- `TabView` / `TabbedPanel`: kontener z tab bar i aktywnym panelem.

## Zakres

- Showcase utrzymac jako preview + boczny panel sterowania, bez prawego tabbed inspectora.
- Wybieranie presetow w showcase zrobic jako `Dropdown` / select input, nie jako `OptionSet`, bo lista presetow ma rosnac do 30+.
- Rozbudowac editor o taby: `Presets`, `Emission`, `Color`, `Size`, `Motion`, `Forces`, `Spawn`, `Burst`, `Export`.
- Dodac `ColorPickerRgb`: kontrolka RGB w stylu Paint, z trzema kanalami `R/G/B`, swatchem i `on_change`.
- Dropdown po rozwinieciu ma dzialac jako popup/overlay ponad layoutem, a nie zwiekszac wysokosc kontrolki i przesuwac widok.
- Utrzymac katalog presetow `mods/playground-2d-particles/presets/` jako przyszly source of truth.
- Katalog powinien obejmowac obecny zestaw showcase: `fire`, `smoke`, `sparks`, `magic`, `snow`, `dust`, `thruster`, `plasma`, `portal`, `rain`, `explosion`.
- Dodac brakujace API tam, gdzie blokuje UX: `set_max_particles`, `set_color_ramp`, `set_shape`, export helper.
- Dopiero pozniej dodac natywne UI `TabView` i `GroupBox`.

## Definition of Done

- Showcase ma dropdown presetow, stabilny preview i nie ma prawego panelu zakladek.
- Editor moze zmieniac emission/color/motion/forces/spawn/burst w odseparowanych panelach.
- Editor ma RGB color picker i moze nim zmieniac kolor emitera runtime.
- Editor ma MVP ColorRamp controls: preset dropdown, interpolation dropdown, 4-stop preview i export YAML stopow.
- Rozwiniety dropdown renderuje sie nad innymi kontrolkami i nie zmienia layoutu dokumentu.
- Presety `fire`, `smoke`, `sparks`, `magic`, `snow`, `dust`, `thruster`, `plasma`, `portal`, `rain`, `explosion` sa widoczne z UI.
- `Export` wypisuje wrapper `particle-preset-2d` z `ParticleEmitter2D`, `color_ramp`, `forces`, `spawn_area`, shape i curves.
- Dokumentacja wskazuje, ktore czesci sa kompozycja na obecnym UI, a ktore wymagaja natywnego `TabView`.

## Follow-up

Ostatni krok do usuniecia duplikacji presetow:

- dodac registry/loader `particle-preset-2d` dla `mods/playground-2d-particles/presets/*.yml`;
- showcase powinien miec jeden `preview-emitter` i aplikowac preset z registry;
- obecne ukryte emitery-template'y w `scene.yml` sa tylko etapem przejsciowym po `world.particles.copy_config(source, target)`;
- po loaderze usunac template emitters z `scene.yml` i generowac dropdown options z registry.
