# Task 011 Summary: Retro Vector Asteroids Redesign

## Goal

Turn `playground-2d-asteroids` from a technical playground into a small retro vector arcade game with a real game flow: menu, procedural waves, score, lives, game over, session highscores, LOW mode, minimal HUD, and a thinner Rhai script.

## Scope

- [ ] Start scene in a YAML-defined main menu.
- [ ] Add menu items: `START`, `HIGHSCORES`, `LOW`, `QUIT`.
- [ ] Replace debug/playground HUD with minimal retro HUD.
- [ ] Hide/remove normal-mode debug grid and debug labels.
- [ ] Start ship at screen center.
- [ ] Make shield last exactly `3.0s`.
- [ ] Ensure shield ignores ship-hit gameplay collisions.
- [ ] Temporarily disable or strongly reduce thrust visual/audio.
- [ ] Replace semantic asteroid entities with neutral pool members `asteroid-01..asteroid-16`.
- [ ] Generate waves procedurally from pool members.
- [ ] Remove hardcoded `wave 2`, `asteroid_names`, and `asteroid_alive` from Rhai.
- [ ] Add session highscores screen.
- [ ] Add LOW mode as low-intensity difficulty.
- [ ] Keep all engine work game-agnostic.

## Expected Engine/API Gaps

- [ ] `world.pools.acquire(pool)`
- [ ] `world.pools.release(pool, entity)`
- [ ] `world.pools.release_all(pool)`
- [ ] `world.pools.active_count(pool)`
- [ ] `world.physics.set_circle_radius(entity, radius)`
- [ ] `world.entities.set_property_int/string(...)`
- [ ] `world.motion.reset_freeflight(entity, rotation)`
- [ ] Optional: `world.app.quit()`

## Definition Of Done

- [ ] Game starts at menu.
- [ ] Menu is YAML `UiDocument`.
- [ ] Selected menu item is highlighted.
- [ ] `START` launches gameplay.
- [ ] Ship spawns at `(0, 0)`.
- [ ] Shield lasts exactly `3.0s`.
- [ ] Asteroids come from neutral `asteroids` pool.
- [ ] Waves are procedural.
- [ ] No manual `wave 2`.
- [ ] No `asteroid_alive`.
- [ ] No wave-defining `asteroid_names`.
- [ ] Score/lives/wave/game-over/highscores work.
- [ ] No `Asteroids*` engine systems.
